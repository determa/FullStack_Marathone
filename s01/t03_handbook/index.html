PROTOCOL

HTTP methods


GET
The GET method requests a representation of a resource. Queries using this method can only retrieve data.

HEAD
HEAD requests a resource in the same way as a GET method, but without a response body.

POST
POST is used to send entities to a specific resource. Often causes a state change or some side effects on the server.

PUT
PUT replaces all current resource representations with the request data.

DELETE
DELETE deletes the specified resource.

CONNECT
CONNECT establishes a "tunnel" to the server specified by the resource.

OPTIONS
OPTIONS is used to describe options for connecting to a resource.

TRACE
TRACE invokes the returned test message from the resource.

PATCH
PATCH is used to partially modify a resource.
